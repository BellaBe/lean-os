# Problem-Solution-Specifics Structure

Template for all sales narratives.

## Problem Section

**Purpose**: Make status quo painful

**Include**:
1. Current state description (from discovery)
2. Quantified impact (time, money, opportunity cost)
3. Competitive/strategic risk
4. Why now matters

**Avoid**:
- Generic industry pain
- Assumptions not validated in discovery
- Fear-mongering without data

## Solution Section

**Purpose**: Show how we solve their specific pain

**Include**:
1. Direct mapping: pain â†’ solution
2. Unique value proposition
3. Why us vs alternatives
4. Risk mitigation

**Avoid**:
- Feature dump
- Talking about us vs their problem
- Overpromising

## Specifics Section

**Purpose**: Provide proof and concrete next steps

**Include**:
1. Relevant metrics/proof points
2. Case study (if available)
3. Implementation details (timeline, effort)
4. ROI model or technical specs (persona-dependent)

**Avoid**:
- Irrelevant case studies
- Vague "success stories"
- Missing implementation reality
